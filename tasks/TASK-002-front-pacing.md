# Decide on front pacing and lifecycle strategy

## What
Both fronts resolve before turn 25. The designer should decide how to maintain
narrative pressure across a longer play session.

## Options to evaluate
1. Increase countdown values (simplest — 2-3x current values)
2. Add a third front
3. Implement front chaining (resolving one front activates another)
4. Add a separate "world event" system that fires periodically

## Why
Without active fronts, the simulation runs but produces no escalating consequences.
The "living world" pillar degrades after turn 22.

## Acceptance Criteria
- Designer documents the chosen approach and rationale
- If new content is needed: task created for worldbuild
- If new mechanics needed: task created for architect
